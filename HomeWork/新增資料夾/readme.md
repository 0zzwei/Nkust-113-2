# **Git & GitHub 版本控制與協作練習**

## **目標**

- 熟悉 Git 的基本操作（commit、branch、merge、pull request、conflict resolution 等）。
- 學會 GitHub 上的協作流程，包括 Fork、Pull Request (PR) 與 Merge。
- 個人練習側重於 PR 提交與 Git 操作，團體練習則強調團隊協作與衝突解決。

---

## **練習方式**

### **個人練習**

- 由講師 **建立 GitHub Repository**，學生直接 *Fork* 後 **Clone** 至本地端進行練習。
- 學生須在該 Repo 上 **建立新分支** 進行修改，並透過 **Pull Request (PR) 提交變更**。
- 講師檢查 PR 是否符合要求，學生不需進行 Code Review。

### **團體練習**

- 由講師 **建立 GitHub Repository**，組長 **Fork** 此 Repo，組員 Clone Fork 後的 Repo。
- **每位組員負責不同的部分**，在自己的 **分支開發**，然後合併到組長的 Repo。
- 團隊內需協作並處理 **Merge Conflicts**（衝突解決）。
- 組長最後將整合後的版本 **提交 Pull Request (PR) 給講師的 Repository**，由講師檢查。
